The assignee limitation exists so large issues are broken down into smaller, more manageable pieces. If multiple people are needed, each should have their own issue, or subtask, to track their work. A single ranking strategy is needed. The ranking system is called LexoRank. It uses a combination of numbers and letters from maximum ordering flexibility. An issue's order is stored as a string in a field called rank. Issue order can also be controlled with JQL. Use the Order By clause to sort issues in ascending or descending rank.
