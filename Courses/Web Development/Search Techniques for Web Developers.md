#### [Specifications vs. implementations](https://www.linkedin.com/learning/search-techniques-for-web-developers/specifications-vs-implementations?u=76281980)

> [!transcript]- Transcript
>
> **[0:01](https://www.linkedin.com/learning/search-techniques-for-web-developers/specifications-vs-implementations?u=76281980&t=1)** - [Instructor] I've used the terms specification and implementation a few times now, so it's important to understand what these terms refer to because it helps understand how the web platform works and why different resources give us different information. For [[HTML]], CSS, [[JavaScript]], and all other web technologies, there are groups dedicated to creating standards specifications. These specifications are verbose and ideally unambiguous descriptions of how a feature should behave when encountered on the web. That's because in the early days of the web, there were no set specifications, so each browser would make up their own interpretations of web code and would often even introduce their own code into the platform. This created a situation where some sites only worked in one browser, which ran counter to the whole intent of the web. This is what we often refer to as the browser wars. The Worldwide Web Consortium, W3C for short, took on the role of defining standard specifications for the web, with the exception of JavaScript, which is handled by TC39. The goal was to create a single source of truth for how web technologies should be implemented. That way, a web feature should function the same no matter what user agent or browser you use. Now, whether that specification is actually implemented is up to the implementers, the browsers and other user agents. For each feature specification, browser vendors can choose to implement it or not. That decision is often made based on the paving the cowpaths principle, meaning when a lot of developers start using a feature in a specific way. They lay down a cowpath, and then the browsers will pave that cowpath
>
> **[1:34](https://www.linkedin.com/learning/search-techniques-for-web-developers/specifications-vs-implementations?u=76281980&t=94)** by implementing that feature more broadly. The thing to remember here is browsers implement specifications, but not all specifications are implemented by browsers. So even if something is documented by the W3C specification, there's no guarantee it'll actually work in any browser. An example of this is the SVG feComposite filter. The specification exists, and it's quite elaborate. It even has a code example at the bottom where they display of what it should look like in browsers. So what does this mean for you as you search for reliable information? In most cases, the best place to search is the platform resources, like MDN Web Docs, because they focus on implementation. Now, if you find a feature there, meaning it is implemented, and you still need more information, maybe to get a fuller understanding of the intended use of the feature or to check out the specification itself, then go to W3C. The simple rule of thumb here is specifications are mainly for browser manufacturers, implementation documentation is for everyone else.

> [!transcript]- Transcript
>
> **[0:00](https://www.linkedin.com/learning/search-techniques-for-web-developers/mastering-search?u=76281980&t=0)** - [Instructor] A big part of finding the right answer to your query is knowing where to search and how to search. In the previous chapter, I introduced the source pyramid to provide some guidance on where to search for information. Now, we'll turn to the question of how to perform a successful search. The key here is understanding how search systems work and how to get them to give you the results you're looking for. The challenge with any type of internet search is there is so much information out there and everyone does everything in their power to drive you towards their information because the internet economy runs on visits and engagement. Search engines use advanced [[Algorithms]] to try to guide you towards the most credible sources of information for what you're looking for, and they also try to make sense of your searches to give you the most relevant information. Even so, their search results are often not quite as accurate as you'd want them to be, and you'll need to do some more work to get the information you actually need.
