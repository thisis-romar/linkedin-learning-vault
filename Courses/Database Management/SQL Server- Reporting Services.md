> **[0:00](https://www.linkedin.com/learning/sql-server-reporting-services-24671486/what-is-sql-server-reporting-services-ssrs?u=76281980&t=0)** - [Instructor] When it comes time to present the data that you have stored in your [SQL Server](../../Skills/Software%20Development/Microsoft%20SQL%20Server.md) [Databases](../../Skills/Software%20Development/Databases.md), you need a way to organize and visualize that data. This is what SQL Server Reporting Services does. So what do I mean by present the data? Well, when you run a select query in a database, you get back results in the form of a table. And while you can export that table to a spreadsheet or take a screenshot of it, it's not really a great way to share that information with others. Instead, it would be much better to spend some time formatting that data to make it more legible. Perhaps integrate it with a chart or other graphical representation so that the meaning of the data can be understood at a glance. You can also give the data more context with some explanatory titles and callouts, and allow users to drill down from a high level summary overview into the nitty gritty details and individual data points. What I've just described is a report. A document that allows you to better share your data with others and allows them to quickly understand its meaning. Reporting Services is a collection of add-on tools and services that build upon SQL Server, and help you build these kinds of custom reports. The main component is called Report Builder. This is a [Windows](../../Glossary/Service/Windows.md) application that works much like a regular [Microsoft Office](../../Skills/Web%20Development/Microsoft%20Office.md) page layout program like Publisher or [PowerPoint](../../Skills/Cloud%20Computing/Microsoft%20PowerPoint.md). With Report Builder, you'll be able to define the look and feel of your reports and pull up to the minute data from your SQL Server databases.
>
> **[1:34](https://www.linkedin.com/learning/sql-server-reporting-services-24671486/what-is-sql-server-reporting-services-ssrs?u=76281980&t=94)** The database connection details, layout and formatting are bundled together into a report definition. And is made available to users, so that they can run your report and view the content. In order to access your reports, users will go through the other component of reporting services, which is a personalized web portal. The Report Server portal is installed on top of a SQL Server instance, and it'll create its own database to manage the reports that you've saved as well as user access permissions. Once logged into the portal, users can choose to view predefined page needed reports, and high level metrics called Key Performance Indicators or KPIs. They can quickly reveal the most important things to know about the state of your business. Users with appropriate permissions can even launch Report Builder right from the portal to start laying out their own reports. So that's what Reporting Services is. It's a collection of tools that enhance the data that you have stored in a SQL Server database, and allows you to collect and share data in a more visual way. This can lead to better insights and help service actionable information.
