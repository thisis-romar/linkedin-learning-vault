>
> **[1:35](https://www.linkedin.com/learning/openai-api-introduction/5-steps-to-using-the-openai-api?u=76281980&t=95)** In the playground, you have full control over every aspect of the API.

> **[1:39](https://www.linkedin.com/learning/openai-api-introduction/5-steps-to-using-the-openai-api?u=76281980&t=99)** The UI lets you set the model you use, the temperature and all the parameters governing the output, and importantly, the maximum length, the maximum number of tokens the API is allowed to use for each response.
>
> **[1:51](https://www.linkedin.com/learning/openai-api-introduction/5-steps-to-using-the-openai-api?u=76281980&t=111)** If you're new to the OpenAI API, I recommend you start here in the playground to familiarize yourself with how the AI responds to different inputs, what the model and parameter settings do, and to set up and manage your own custom assistance.
>
> **[2:04](https://www.linkedin.com/learning/openai-api-introduction/5-steps-to-using-the-openai-api?u=76281980&t=124)** The playground also helps you write code for basic integration into your apps.
>
> **[2:08](https://www.linkedin.com/learning/openai-api-introduction/5-steps-to-using-the-openai-api?u=76281980&t=128)** Once you've configured your chat settings, click the code button and you get the configuration settings in Python, Node.js, cURL, and even JSON, so you can just copy that code and paste it into your app and start working right away.
>
> **[2:21](https://www.linkedin.com/learning/openai-api-introduction/5-steps-to-using-the-openai-api?u=76281980&t=141)** That brings us to item number four, authentication.
>
> **[2:24](https://www.linkedin.com/learning/openai-api-introduction/5-steps-to-using-the-openai-api?u=76281980&t=144)** The OpenAI API uses API keys for authentication.
>
> **[2:27](https://www.linkedin.com/learning/openai-api-introduction/5-steps-to-using-the-openai-api?u=76281980&t=147)** You can generate and manage unique API keys from platform.[openai.com](https://openai.com), and from here you can set up as many API keys as you like.
>
> **[2:35](https://www.linkedin.com/learning/openai-api-introduction/5-steps-to-using-the-openai-api?u=76281980&t=155)** To create a new API key, click the Create new secret key button, give your key a descriptive name, and copy it once it appears in the modal popup.
>
> **[2:43](https://www.linkedin.com/learning/openai-api-introduction/5-steps-to-using-the-openai-api?u=76281980&t=163)** One important thing to keep in mind here, you only get to see this key once in this modal, so make sure you copy it before you close the model.
>
> **[2:51](https://www.linkedin.com/learning/openai-api-introduction/5-steps-to-using-the-openai-api?u=76281980&t=171)** Now, if you accidentally close the modal, don't worry.
>
> **[2:54](https://www.linkedin.com/learning/openai-api-introduction/5-steps-to-using-the-openai-api?u=76281980&t=174)** Just delete the key and make a new one.
>
> **[2:56](https://www.linkedin.com/learning/openai-api-introduction/5-steps-to-using-the-openai-api?u=76281980&t=176)** They are effectively throw away strings of random text In your code, the API key is passed to the API as a bearer token in the authentication header of your request.
>
> **[3:05](https://www.linkedin.com/learning/openai-api-introduction/5-steps-to-using-the-openai-api?u=76281980&t=185)** This means you have to be careful where you store the API key in your application.
>
> **[3:10](https://www.linkedin.com/learning/openai-api-introduction/5-steps-to-using-the-openai-api?u=76281980&t=190)** For example, in a standard browser-based application, the key will be stored in code, meaning it's visible to other users.
>
> **[3:17](https://www.linkedin.com/learning/openai-api-introduction/5-steps-to-using-the-openai-api?u=76281980&t=197)** This is not secure and not recommended.
>
> **[3:20](https://www.linkedin.com/learning/openai-api-introduction/5-steps-to-using-the-openai-api?u=76281980&t=200)** So if you're building a web-based application using the OpenAI API, the best practice is to do the actual AI integration on the backend from the server and have your front-end application send its request through that backend.
>
> **[3:33](https://www.linkedin.com/learning/openai-api-introduction/5-steps-to-using-the-openai-api?u=76281980&t=213)** Speaking of development, fifth and finally, code integration.
>
> **[3:36](https://www.linkedin.com/learning/openai-api-introduction/5-steps-to-using-the-openai-api?u=76281980&t=216)** The OpenAI API is a standard RESTful API that acts on HTTP requests.
>
> **[3:41](https://www.linkedin.com/learning/openai-api-introduction/5-steps-to-using-the-openai-api?u=76281980&t=221)** That means you're free to integrate it using whatever coding language and REST-based approach you prefer.
>
> **[3:47](https://www.linkedin.com/learning/openai-api-introduction/5-steps-to-using-the-openai-api?u=76281980&t=227)** That said, OpenAI has two robust libraries for the API to make integration easier.
>
> **[3:52](https://www.linkedin.com/learning/openai-api-introduction/5-steps-to-using-the-openai-api?u=76281980&t=232)** One for Python and one for Node.js and TypeScript.
>
> **[3:56](https://www.linkedin.com/learning/openai-api-introduction/5-steps-to-using-the-openai-api?u=76281980&t=236)** Both of these libraries are simply called OpenAI and can be added as dependencies to your project using pip for Python and npm or NPX for Node and TypeScript.
>
> **[4:05](https://www.linkedin.com/learning/openai-api-introduction/5-steps-to-using-the-openai-api?u=76281980&t=245)** The OpenAI libraries provide helper functions to simplify integration and copious documentation, including practical code examples to get you started.
>
> **[4:13](https://www.linkedin.com/learning/openai-api-introduction/5-steps-to-using-the-openai-api?u=76281980&t=253)** To get started using the OpenAI libraries in either Python or JavaScript, import the library, create a new instance of OpenAI and pass in your API key.
>
> **[4:22](https://www.linkedin.com/learning/openai-api-introduction/5-steps-to-using-the-openai-api?u=76281980&t=262)** From here, you're now free to use all the features of the OpenAI API programmatically in any way you want.
>
> **[4:27](https://www.linkedin.com/learning/openai-api-introduction/5-steps-to-using-the-openai-api?u=76281980&t=267)** To help you get started, the exercise files for this course holds code example for the basic setup of the API using Python and Node.js and a more in-depth example of how you can use the streaming feature to create an interactive application.
>
> **[4:41](https://www.linkedin.com/learning/openai-api-introduction/5-steps-to-using-the-openai-api?u=76281980&t=281)** The README file in the GitHub repository explains how to set up and use these files in GitHub code spaces.
>
> **[4:47](https://www.linkedin.com/learning/openai-api-introduction/5-steps-to-using-the-openai-api?u=76281980&t=287)** And that's it, these are the five main things you need to know to get started working with OpenAI API.
