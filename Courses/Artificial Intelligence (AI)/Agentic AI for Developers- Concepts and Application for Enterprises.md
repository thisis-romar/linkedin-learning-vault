#### [Tool use pattern in agentic AI](https://www.linkedin.com/learning/agentic-ai-for-developers-concepts-and-application-for-enterprises/tool-use-pattern-in-agentic-ai?u=76281980)

> [!transcript]- Transcript

This is arguably the most popular pattern in agentic AI. In the Tool Use pattern, the LLM is used to decide if and when a tool should be used. It also decides on how to use the tool, specifically what inputs are provided to the tool. How does Tool Use work? Tools are like actions or routes. Unlike routes, tools also have input parameters associated with them that needs to be currently identified and passed on to them. These tools may include data sources, functions, retrieval techniques, and system integrations. Like routes, tools also have associated profiles about their capabilities and inputs required to use the tools. The agent is given a goal. The agent then uses the LLM to decide on how to execute the goal. The LLM may decide to use one or more tools called in a specific sequence to achieve the goal. In addition to deciding on the tool, the LLM also need to figure out the exact parameters that need to be passed when calling the tool. The value of these inputs may be derived from the goal itself or from the context history. The agent executes the tool with the chosen inputs. In general, this process is similar to [[Routing]] except that the LLM now also data mines the parameter values when calling the tool.

Goals in agents may require a multi-step workflow to achieve its end results. Planning uses an LLM to break down a goal into a series of subgoals. It then maps subgoals to tasks and subtasks. These tasks then get executed as a workflow. Why do we need planning? A goal in an AI agent can be complex. It will need to be broken down into multiple tasks with the output of one task becoming the input to the other task. Each task or step would then need its own logic to execute. The planning pattern is used to take a goal, break it down into a series of subgoals, tasks, and subtasks, such that each task can then be executed with an available action or tool. The agent uses the LLM to do this planning. The LLM analyzes the goal, the available tools and actions to determine a workflow through which the goal can be achieved. Planning will leverage the other patterns like reflection, [[Routing]], tool use, and multi-agent interactions to achieve the tasks. > **[0:00](https://www.linkedin.com/learning/agentic-ai-for-developers-concepts-and-application-for-enterprises/hallucinations-in-agentic-ai?u=76281980&t=0)** - [Instructor] Having seen agentic AI in action, let's also discuss some key considerations when building agents for enterprises in this chapter. We begin with hallucinations. LLMs are well known for their tendency to hallucinate. When LLM makes decisions in agentic AI, it can also hallucinate. Because agents are autonomous, they will proceed to execute on the hallucinations, too, and this can lead to significant and sometimes disastrous results. So how do we mitigate hallucinations in agentic AI? When choosing LLMs for agents, it is important to evaluate them for the specific use cases and measure their accuracy. The [[Probability]] to hallucinate should also be evaluated. When an agent is in production, it's recommended to have [[Continuous Monitoring]] of the agent's performance, especially with respect to accuracy and hallucinations. Hallucination guardrails are available, and those should be used to validate the outputs of the LLMs. In addition, we can use the reflection pattern. We can add a critique LLM to the flow to evaluate if the plans generated by the original LLM are correct. For critical workflows, it is recommended to have a human in the loop to inspect plans and outputs generated by the agent, and approve them before they are executed.

A continuous learning loop with human feedback will help the LLM understand its mistakes and fine-tune its performance over time. It's also recommended to have periodic audits for the performance of agents.
