> **[0:00](https://www.linkedin.com/learning/iot-foundations-standards-and-ecosystems-14427225/iot-architectures?u=76281980&t=0)** - [Instructor] The architecture for [IoT](../../Glossary/Concept/IoT.md) system is based on the Open Systems Interconnection or OSI model. The OSI model was introduced by the International Organization for Standardization or ISO, in 1984. It is a seven layer model, physical, data, link, network, transport, session, presentation, and application. Each layer provides services to the layer up or down the stack. OSI is a reference model of how components can communicate over database networks. Usually, the IOT architecture is divided into subsystems. The generic model has three layers, physical, communication, and application. Physical layer consists of the [Hardware](../../Topics/Hardware.md) objects to sense the status of things such as sensors, actuators, human operators, computers, and smartphones. The second layer is the communication and network layer, includes the different communication methods and standards that are applied between the devices and the backend services. The third layer is the application and service layer. This includes examples like event processing of the data,
>
> **[1:37](https://www.linkedin.com/learning/iot-foundations-standards-and-ecosystems-14427225/iot-architectures?u=76281980&t=97)** healthcare management, and environmental monitoring. These three layers are considered by the I triple E P 2 4 1 3 standard as the fundamental IoT architecture and can be used as a reference model for any actual IoT system. Here's an example, the first layer where data is collected, then, using communication to transfer the data, and finally, for data analysis. In the IoT architecture model, the data flows in both directions, from the physical to application layers and vice versa. A complete IoT architecture has a flawless operation of components and reliability within its design. The IoT architecture is unique for each system. It depends on the different devices utilized, the communication methods, and the services and applications the system is used for. The IoT architecture is adjusted accordingly while still including the main three layers' subsystems. Let's take the example of smart transportation: the physical layer, including the sensors within the car, the communication layer is the internet, and the application layer is the GPS system or the data analysis.
