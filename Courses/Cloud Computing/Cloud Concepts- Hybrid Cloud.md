We will be discussing cloud deployment models. Including their definition, benefits and use cases. By breaking down these concepts. My goal is to help you understand and differentiate which deployment model would be best for your organization. Deployment models in cloud computing refer to the type of architecture a cloud solution will be implemented on. Currently, there are a couple of main cloud deployment models you can choose from. They are public cloud, private cloud, community cloud hybrid cloud, and multicloud. Let's break down each one of these models. Public cloud computing is one of the more popular models that people are familiar with. Organizations utilizing public cloud architecture deploy their data and applications on infrastructure that is managed by a third-party vendor. Meaning that the infrastructure in this model is shared amongst multiple organizations using the public internet. Typically to get started, one simply needs to browse to a public cloud provider. For example, the [[Google Cloud Platform (GCP)|Google cloud platform]] and create an account and voila, you have access to cloud computing resources. Benefits to using this model include minimal upfront costs, high [[Scalability]] and increased reliability for an organization.
>
> **[1:34](https://www.linkedin.com/learning/cloud-concepts-hybrid-cloud/defining-cloud-concepts?u=76281980&t=94)** Experienced challenges with this model includes security and [[Privacy]] concerns, lack of customized solutions and a lack of control. Next is private cloud. Whereas public cloud computing was shared amongst organizations, private [[Cloud Infrastructure]] is not. Private cloud solutions are built exclusively for an individual organization with a single company controlling all of the virtualization resources. [[Private Clouds]] can be built right on premise for the organization. So that way, resources are close by, or it can be stored elsewhere with a third-party. Regardless of where it's located, private clouds offer more security for an organization that may need more control over its resources. Additional benefits include support for legacy systems and improved performance. Deterrents for this model include higher costs since you would have to set up and maintain [[Hardware]] resources and limited scalability. The community cloud model is not commonly used by organizations, but it's still important to understand in case you ever come across this type of architecture. Similar to the public cloud model, community cloud infrastructure is shared and jointly accessed by several organizations. The main difference between the two is that community cloud infrastructure is shared amongst organizations, typically within the same industry. Examples of this include universities and government agencies.
>
> **[3:09](https://www.linkedin.com/learning/cloud-concepts-hybrid-cloud/defining-cloud-concepts?u=76281980&t=189)** The three previous models I covered all dealt with singular architecture types. While useful, sometimes these models don't fit every organization's use case. That's where hybrid and multi-cloud computing come into play. Hybrid cloud refers to a mix of private and public [[Cloud Services]] and infrastructure to meet the needs of an organization. For example, having mission critical data on premise and utilizing AWS for certain applications. For many, having all of your data in the cloud is not necessary or even feasible, especially in situations where security and data control is a priority. Having this hybrid mix of services is a model that many organizations are coming to the conclusion of in order to reap the benefits of the cloud while still meeting compliance or regulatory needs. In a later module, we will talk about the benefits and challenges of hybrid cloud. Our last cloud deployment model is multicloud. Similar to hybrid cloud, the multicloud deployment model is where an organization uses two or more cloud platforms to deliver its applications and services. This environment can also be a mix of public or private clouds, but typically involves more than one cloud vendor. Which can also introduce some complexity. Usually this type of model fits organizations when one provider does not have all of their features your different types of users are asking for.
>
> **[4:43](https://www.linkedin.com/learning/cloud-concepts-hybrid-cloud/defining-cloud-concepts?u=76281980&t=283)** Overall, cloud computing offers many different architectural options for organizations to choose from. Now that we have covered all of these models, you'll be able to evaluate which best suits your organization's needs.

#### [Hybrid cloud benefits](https://www.linkedin.com/learning/cloud-concepts-hybrid-cloud/hybrid-cloud-benefits?u=76281980)

>
> **[0:00](https://www.linkedin.com/learning/cloud-concepts-hybrid-cloud/hybrid-cloud-benefits?u=76281980&t=0)** - [Instructor] [[Hybrid Cloud]] computing offers numerous benefits and advantages. One of its first benefits and drivers for this model comes from the cost savings. Cost is a big factor for organizations that are considering migrating workloads to the cloud. While there is an upfront investment to create a hybrid infrastructure, in the long term, it can drastically reduce IT spending. Many organizations are realizing that a purely private or public infrastructure just doesn't fit their long-term needs. Unfortunately, if this is discovered after starting a cloud migration, an organization can find themselves having to switch up their infrastructure setup, which can lead to higher overall costs in [[Cloud Computing]]. By keeping critical data in a private portion of a hybrid environment, and migrating less critical data to various cloud providers, an organization can avoid complications that come with moving data after the fact. A second important benefit comes from the flexibility and [[Scalability]] of hybrid cloud computing. Hybrid clouds can scale to support times of high demand for applications or services without an organization having to over provision its own data centers. Through a concept called cloud bursting, organizations can access additional computing resources for their private cloud without service interruption. This is especially useful, for example, in businesses who might experience seasonal peak demands
>
> **[1:35](https://www.linkedin.com/learning/cloud-concepts-hybrid-cloud/hybrid-cloud-benefits?u=76281980&t=95)** but may not need those extra resources any other time. The ability to scale and combine the current existing private cloud with the public cloud allows businesses to thrive. The third advantage of hybrid cloud environments comes from the need for more control and security over an organization's data. Security is a core component for many enterprises and handing over full control of how your data is transferred, stored and processed can be a major concern. Instead of moving all of your data into a purely public [[Cloud Infrastructure]], businesses can keep their most sensitive data in their own managed data center. Hybrid cloud offers the best of both worlds, allowing enterprises to keep their assets where they can see and control them. Lastly, uniformity is another advantage often overlooked in cloud computing. Standardizing IT management practices can be difficult when you're managing two different deployment models. Luckily, public cloud providers can help enterprises create and implement uniformity across the different environments, allowing enterprises to take advantage of public and private infrastructure. [[Microsoft Azure]] Stack and [[Google]] Anthos both offer hybrid services that enable cross-deployment uniformity. While there are many benefits to hybrid cloud computing, your unique and specific infrastructure needs
>
> **[3:08](https://www.linkedin.com/learning/cloud-concepts-hybrid-cloud/hybrid-cloud-benefits?u=76281980&t=188)** will be the biggest factor.
