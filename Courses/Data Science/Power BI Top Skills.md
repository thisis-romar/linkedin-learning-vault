There are two settings you can adjust as needed: cardinality or cross filter direction. Cardinality tells PowerBI how the data from one table connects
>
> **[1:33](https://www.linkedin.com/learning/power-bi-top-skills-27810109/manage-relationships-and-cardinality-in-power-bi?u=76281980&t=93)** to the data to another table. It defines whether one or many values match across tables. In this situation, the system recommends many to many, meaning that multiple roles in one table match multiple roles in another, like employees working on several projects, and employees having multiple instructor led trainings. Cross filter direction, both, means filters apply to both direction between the two tables. If you filter one table, the connected table filters two and vice versa. Use caution with many to many relationships, because they can create ambiguous results and make your visuals misleading. They often allow duplicate values to connect across tables, which can cause incorrect totals or unexpected filtering behavior.
